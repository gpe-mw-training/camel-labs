2. JBoss Developer Studio Overview

JBoss Developer Studio (JBDS) is an integrated development environment (IDE) that integrates and certifies both tooling and runtime components by combining Eclipse, best-of-breed open source tools, and the JBoss Enterprise platform. You must have JBoss Developer Studio installed in your local development environment. Among many other uses, JBDS will be used in these lab instructions to manage your remote OpenShift enabled lab application(s), to develop and design Apache Camel Routes. It will also be used to connect to your remote application via ssh.

JBDS can be downloaded from the jboss.org web site and the procedure to install it is described here on the RedHat Customer Portal

JBDS includes a variety of different plugins. The following is a list of those JBDS plugins that are of needed for execution of your GPE sponsored middleware course labs:

Integration Stack Suite

The Integration Stack Suite of plug-ins is of particular importance for the JBoss Fuse & JBoss AMQ middleware courses. The Integration Stack Suite of plug-ins are not included out-of-the-box with JBDS. Instead, the suite needs to be manually installed following the procedures in the JBDS IntegrationStack documentation.

3. Install Fuse Tooling

  • Start JBoss Developer Studio 8

    • Accept the default workspace

    • Close the welcome windows

  • In the JBoss Central main window, click the bottom tab Software/Update

  • In the bottom right, click Enable Early Access

  • Click JBoss Integration and SOA Development

  • At the bottom of the window, click Install/Update.

  • Click Deselect All

  • Only select the Fuse components. Your screen should resemble the one below.

  • Click Next

  • Accept the remaining defaults and click thru to complete the installation process.

    • Note: The installation will take a while since it has to download components from the Internet.

  • Once installation is complete, you will need to restart JBoss Developer Studio

4. Confirm Fuse Tooling Installation

  • Confirm the Fuse Tooling Installation

    • Select File > New > Other > Fuse Tooling > Fuse Project

    • For project name, type: cameldemo

    • Click Next

    • In the list of artifacts, choose Artifact ID camel-archetype-spring

  • Accept all other defaults and click Finish

  • Wait for about 20 seconds for the project to build completely (monitor with the status bar in the bottom right-corner).

  • Expand the sub-folder, Camel Context and double-click the file camel-context.xml
